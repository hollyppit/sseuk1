# 슥_미술 독학 솔루션 프로젝트 계획

## 프로젝트 개요
- **프로젝트명**: 슥_미술 독학 솔루션
- **목적**: 미술 독학을 위한 온라인 솔루션 제공. 튜토리얼, AI 기반 피드백, 커뮤니티 기능을 통해 사용자들이 미술을 쉽게 배울 수 있도록 지원.
- **대상 사용자**: 미술 초보자, 독학 희망자, 취미 미술가.
- **수익 모델**: 유료 튜토리얼 (건당 ₩1,500), 프리미엄 AI 분석, 광고 수익.

## 플랫폼 구조 (CLAUDE.md 기반)
- **웹 버전 (슥_미술 독학 솔루션-web/)**: 브라우저 기반, 순수 HTML/JS 구조. npm 빌드 없음. Cloudflare Functions (API) 포함. 환경변수 관리.
- **토스 미니앱 (슥_미술 독학 솔루션-toss/)**: Vite + @apps-in-toss/web-framework. UI 전용, API는 웹 버전 절대경로 호출. 빌드 후 .ait 파일 토스 콘솔 업로드.
- **앱 버전 (슥_미술 독학 솔루션-app/)**: React Native (추후 구현). API는 웹 버전 호출.

## 개발 규칙
- 작업 전 폴더 역할 확인: 각 플랫폼 폴더 외 건드리지 말 것.
- API: 웹 버전에만 추가, 다른 버전은 절대경로 호출.
- 환경변수: 웹 버전 Cloudflare Pages에만 등록 (예: ANTHROPIC_API_KEY, GEMINI_API_KEY 등).
- 배포: 웹은 git push → Cloudflare 자동. 토스는 빌드 → 토스 콘솔 업로드.

## 기능 요구사항
1. **튜토리얼 제공**: 단계별 미술 강의 (기초, 중급, 고급).
2. **AI 피드백**: 사용자가 업로드한 그림을 AI (Claude, Gemini)로 분석 및 피드백 제공.
3. **커뮤니티**: 사용자 작품 공유, 댓글, 좋아요 기능.
4. **결제 시스템**: 토스페이 연동 (유료 콘텐츠 구매).
5. **로그인**: 토스 로그인 연동.
6. **광고**: 무료 콘텐츠에 광고 삽입.

## 개발 단계 계획
### Phase 1: 기초 구축 (완료)
- 웹 버전 기본 구조 설정 (HTML/JS) ✓
- 디자인 테마 유지하면서 'sseuk' 서비스에 맞게 텍스트, UI, 레이아웃 개선 ✓
- GitHub 레포 연결 및 초기 배포 ✓

### Phase 2: 핵심 기능 개발 (2-4주)
- 튜토리얼 페이지 구현.
- AI 피드백 기능 (이미지 업로드, 분석 API 호출).
- 커뮤니티 기능 (CRUD API).

### Phase 3: 결제 및 로그인 통합 (1-2주)
- 토스 로그인 연동.
- 토스페이 결제 시스템 구현.

### Phase 4: 토스 미니앱 개발 (1-2주)
- 슥_미술 독학 솔루션-toss/ 폴더 생성.
- Vite 프로젝트 설정, @apps-in-toss/web-framework 설치.
- 웹 버전 API 호출로 UI 구현.

### Phase 5: 테스트 및 배포 (1주)
- 단위 테스트, 통합 테스트.
- 웹 버전 배포 (Cloudflare).
- 토스 미니앱 빌드 및 업로드 (토스 콘솔).

### Phase 6: 고도화 (지속적)
- 사용자 피드백 수집 및 개선.
- 추가 기능: 실시간 채팅, 고급 AI 모델, 모바일 앱 개발.
- 성능 최적화, 보안 강화.

## 기술 스택
- **프론트엔드**: HTML/JS (웹), React/TypeScript (토스 미니앱).
- **백엔드**: Cloudflare Functions (Node.js).
- **AI**: Claude API, Gemini API.
- **결제/로그인**: 토스 SDK.
- **배포**: Cloudflare Pages, 토스 콘솔.

## 위험 요소 및 대응
- API 키 보안: 환경변수로 관리, 절대 코드에 하드코딩.
- CORS 이슈: 토스 미니앱에서 웹 API 호출 시 Origin 설정.
- 용량 제한: 앱 번들 100MB 이하 유지, 리소스 CDN 사용.
